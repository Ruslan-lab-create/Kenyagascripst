# Roblox Scripts Bot + Trafsly

Telegram-бот с админ-панелью для раздачи Lua-скриптов через короткие ссылки
с обязательной подпиской на спонсоров (Trafsly).

## Как это работает

1. Админ через `/admin` создаёт скрипт → бот выдаёт короткую ссылку вида
   `https://t.me/твой_бот?start=AbC12xYz`.
2. Пользователь переходит по ссылке → бот запрашивает у Trafsly
   (`/get-sponsors`) список заданий на подписку.
3. Если задания есть — показывает каналы с кнопками «Подписаться» + «Я подписался».
4. По нажатию «Я подписался» бот дёргает `/confirm-subscription` для каждого
   задания. Если всё подтвердилось — отдаёт текст скрипта.
5. Если заданий нет (или Trafsly временно недоступен) — скрипт отдаётся сразу,
   без блокировки пользователя.

## Установка

```bash
python3 -m venv venv
source venv/bin/activate       # Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
```

Заполни `.env`:

- `BOT_TOKEN` — токен от [@BotFather](https://t.me/BotFather)
- `BOT_USERNAME` — юзернейм бота без `@`
- `ADMIN_IDS` — твой Telegram ID (узнать у [@userinfobot](https://t.me/userinfobot)), через запятую если админов несколько
- `TRAFSLY_TOKEN` — получишь после шагов ниже

## Подключение Trafsly (по шагам)

1. Открой [@TrafslyBot](https://t.me/TrafslyBot) → раздел «Мои боты» → добавь своего бота.
2. Отправь его **на модерацию** — без этого API-токен работать не будет.
3. После одобрения зайди в настройки бота → **Интеграция** → скопируй API-токен
   (начинается с `at_`).
4. Вставь его в `.env` как `TRAFSLY_TOKEN`.
5. В том же кабинете можно настроить `max_sponsors` (сколько заданий показывать
   за раз) и таргетинг по 21 фильтру — это делается в самом @TrafslyBot,
   бот на своей стороне ничего дополнительно настраивать не должен.

**Важно:** никогда не публикуй `TRAFSLY_TOKEN` в открытом виде (гитхаб, чаты
и т.п.) — по нему можно списывать деньги с твоего баланса от твоего имени.
`.env` уже в `.gitignore`-стиле — не коммить его.

## Запуск

```bash
python3 bot.py
```

Бот запустится в режиме long polling. Для продакшена рекомендуется поднять
на VPS через `systemd` или `screen`/`tmux`, чтобы процесс не падал при
закрытии терминала.

## Структура проекта

```
roblox_bot/
├── bot.py              # точка входа
├── config.py            # загрузка .env
├── database.py           # SQLite: скрипты, сессии проверки подписки
├── trafsly_api.py        # обёртка над Trafsly API
├── keyboards.py           # инлайн-клавиатуры
├── states.py             # FSM-состояния (создание скрипта)
├── handlers/
│   ├── admin.py          # админ-панель
│   └── user.py           # флоу пользователя (старт по ссылке, ОП)
├── requirements.txt
└── .env.example
```

## Команды

- `/admin` — открыть админ-панель (только для ID из `ADMIN_IDS`)

## Деплой на Railway через GitHub

### 1. Залей проект на GitHub

```bash
cd roblox_bot
git init
git add .
git commit -m "Initial commit"
```

`.gitignore` уже исключает `.env`, `bot.db`, `venv/` — секреты в репозиторий не попадут.
Дальше на github.com создай пустой репозиторий (без README/license) и:

```bash
git remote add origin https://github.com/ТВОЙ_ЮЗЕРНЕЙМ/НАЗВАНИЕ_РЕПО.git
git branch -M main
git push -u origin main
```

### 2. Подключи Railway

1. [railway.app](https://railway.app) → **New Project** → **Deploy from GitHub repo** → выбери свой репозиторий
2. Railway сам определит Python-проект по `requirements.txt` и `Procfile` (тип `worker`)
3. Зайди в **Variables** созданного сервиса и добавь переменные вручную (не из `.env` файла — он не в репозитории):

   | Переменная | Значение |
   |---|---|
   | `BOT_TOKEN` | токен от @BotFather |
   | `ADMIN_IDS` | твой Telegram ID |
   | `BOT_USERNAME` | юзернейм бота без @ |
   | `TRAFSLY_TOKEN` | токен из @TrafslyBot (после модерации) |
   | `DB_PATH` | `/data/bot.db` |

4. **Обязательно добавь Volume** (иначе база скриптов будет удаляться при каждом
   передеплое, т.к. диск Railway по умолчанию эфемерный):
   Service → **Settings** → **Volumes** → **New Volume** → Mount path: `/data`

5. **Settings → Deploy → Start Command** проверь, что стоит `python3 bot.py`
   (обычно подхватывается из Procfile автоматически)

6. Нажми **Deploy**. В логах должно появиться `Бот запускается...`

### 3. Обновление бота в будущем

```bash
git add .
git commit -m "описание изменений"
git push
```

Railway передеплоит автоматически при пуше в `main`.

## Что можно доработать дальше

- Отправка длинных скриптов файлом (`.lua`), а не текстом — сейчас длинные
  скрипты просто обрезаются до 4000 символов с предупреждением.
- Статистика по каждому скрипту через `/statistic` Trafsly прямо в админке.
- Ограничение частоты запросов (rate limit) на случай накрутки переходов.
- Редактирование текста уже созданного скрипта без пересоздания.
